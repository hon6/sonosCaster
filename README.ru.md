Передача системного звука Windows на колонку Sonos в реальном времени.
Всё, что играет на ПК (браузер, видео, музыка, игры), звучит из Sonos.
В режиме низкой задержки замеренное сквозное время составляет **~0,2 с**.

В углу экрана висит плавающий HUD в виде капсулы в стиле macOS —
переключайте вкл./выкл., наведите курсор, чтобы развернуть выбор устройства,
громкость и настройки.

> Изначально проект был нацелен на AirPlay, но современные колонки Sonos
> работают по AirPlay 2 с обязательным шифрованием, которое open-source
> проекты передавать не могут. Перешли на родной для Sonos путь через UPnP:
> обходит шифрование и в итоге даёт даже меньшую задержку.

## Как это работает

```
PC system audio (WASAPI loopback)
   → ffmpeg-less raw PCM with a 4 GB WAV header (low-latency)  *or*  ffmpeg MP3
   → local HTTP audio stream at http://<lan-ip>:8009/stream.wav
   → SoCo tells Sonos to play that URL (auto-resolves current group coordinator)
```

- **Обнаружение**: `soco` (zeroconf), устройство запоминается по стабильному **UID** — переживает перезагрузку роутера.
- **Захват**: `soundcard` (WASAPI loopback — то, что реально играет на ПК).
- **Стриминг**: сырой PCM с трюком WAV-заголовка длиной 4 ГБ = никакой буферизации энкодером. Для случаев с узким каналом доступен путь через MP3 (ffmpeg).
- **Управление**: `soco.play_uri` с динамическим разрешением координатора и повтором запросов, чтобы surround-группы (например, Playbase + 2× Play:One) продолжали работать при смене координатора.

## Задержка

| Режим | Замер | Подходит для |
|------|----------|----------|
| **Низкая задержка (WAV / сырой PCM)** | **~0,2 с** | Видео, музыка, нетребовательные игры |
| MP3 | ~4 с | Только музыка / подкасты (меньше трафика) |

По умолчанию используется путь с низкой задержкой. Порог в 0,2 с — это
собственный буфер мультирум-синхронизации Sonos, уже на пределе прошивки.
Режим WAV даёт ~1,4 Мбит/с — это нормально для любого домашнего WiFi.

## Установка (разработка / запуск из исходников)

Требуется **Python 3.9+** и **ffmpeg** в PATH.

```powershell
# 1. ffmpeg (one-time)
winget install Gyan.FFmpeg

# 2. Clone + enter
git clone git@github.com:hon6/sonosCaster.git
cd sonosCaster

# 3. Virtualenv + deps
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt

# 4. Run
.venv\Scripts\python.exe main.py
```

Или дважды кликните по `run.bat`.

## Сборка автономного .exe

PyInstaller упаковывает Python и ffmpeg в один исполняемый файл размером
60 МБ, который запускается на любом ПК с Windows без установки чего-либо.

```powershell
# Drop ffmpeg.exe into bundle/ first (not tracked in git — too large):
#   download from https://www.gyan.dev/ffmpeg/builds/  ->  bundle/ffmpeg.exe

.venv\Scripts\python.exe build.py
# -> dist/SonosCaster.exe
```

## Использование

1. В углу появляется капсула. Кликните по переключателю — запускается обнаружение и пересылка звука.
2. Наведите курсор на капсулу, чтобы развернуть: выбор устройства, громкость, настройки, закрытие.
3. Капсула запоминает своё положение; перетаскивается мышью. Возле правого края экрана она разворачивается **влево**.
4. При первом запуске может появиться запрос прав администратора для добавления правила брандмауэра на порт 8009 (чтобы Sonos мог забирать поток). Разрешите один раз.

## Решение проблем

**Sonos не найден** — ПК и Sonos должны быть в одной LAN / WiFi, без изоляции точки доступа.

**Подключено, но тишина** — убедитесь, что (a) на ПК прямо сейчас действительно что-то играет (loopback ловит только живой звук), (b) устройство вывода по умолчанию совпадает с тем, что играет (настройки → источник звука), (c) брандмауэр пропускает порт 8009.

**Surround-группа / ошибка «must be called on the coordinator»** — встроенный механизм повторов с этим справляется; если не восстановилось, выключите и включите один раз.

**Sonos перезагрузился, IP сменился** — работает автоматически; устройства отслеживаются по стабильному UID, а не по IP.

**Хочется ещё меньшую задержку** — 0,2 с — это собственный буфер Sonos. Чтобы получить меньше, нужен другой транспорт (низколатентный BT-передатчик, проводной выход).

## Структура проекта

```
.
├── main.py                  # Точка входа
├── run.bat                  # Запуск в один клик
├── build.py                 # Сборщик .exe через PyInstaller
├── requirements.txt
├── LICENSE
├── README.md
├── bundle/
│   └── ffmpeg.exe           # (скачиваете сами — нет в git)
└── sonos_caster/
    ├── capsule.py           # UI плавающей капсулы (Tkinter + рендер через PIL)
    ├── sonos_caster.py      # Обнаружение, разрешение координатора, play_uri, watchdog
    ├── http_stream.py       # Локальный HTTP-поток + пути WAV/MP3
    ├── capture.py           # WASAPI loopback
    ├── render.py            # Сглаженные UI-примитивы на PIL
    ├── icon.py              # Иконка приложения (буква «S» в стиле Sonos)
    ├── ffmpeg_util.py       # Поиск ffmpeg (PATH / бандл PyInstaller)
    ├── firewall.py          # Помощник для входящего правила на порт 8009
    ├── sysvolume.py         # Чтение/запись мастер-громкости для приглушения при пересылке
    ├── config.py            # Сохранение настроек
    ├── autostart.py         # Помощник для автозапуска через реестр Windows
    └── diagnostics.py       # Встроенный диагностический отчёт (в приложении)
```
